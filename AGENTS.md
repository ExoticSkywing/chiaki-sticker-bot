# Agent 工作规则

本项目使用“阶段性交付包框架”。

工具适配：`generic`

## 核心契约

1. 甲方不满意时，继续推进、修正，并做力所能及的验证；
2. 人类更好操作或更适合判断的验证，例如前端交互、浏览器操作、视觉观感、业务判断，应明确交给人类；
3. 甲方满意时，生成阶段性交付包；
4. 阶段性交付包写入 `docs/phase-deliveries/`；
5. 更新 `docs/HANDOFF.md`；
6. 不把中间迭代过程当成交付物；
7. 不提交 secrets、tokens、runtime 数据、生成缓存。

## 最终汇报必须说明

- 阶段性交付包路径；
- 项目接棒入口是否更新；
- 智能体已执行的验证，以及需要人类执行的验证；
- 当前还有什么后续建议。
