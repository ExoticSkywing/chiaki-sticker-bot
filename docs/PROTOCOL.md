# 阶段性交付包协议

## 核心规则

```text
甲方不满意：继续推进
甲方满意：生成阶段性交付包 + 更新接棒入口
```

## 正式产物

```text
docs/phase-deliveries/  阶段性交付包
docs/HANDOFF.md         项目接棒入口
```

## 辅助材料

```text
docs/evidence/          验证证据，可选
docs/decisions/         长期决策，可选
docs/runbooks/          人工复现步骤，可选
```

辅助材料可以被阶段性交付包引用，但不是主交付物。

## 验证责任

智能体应执行自己力所能及、可稳定运行的验证，例如单元测试、脚本 smoke test、静态检查。

如果验证更适合人类，例如前端交互、浏览器操作、视觉观感、真实业务判断、账号授权流程，应在阶段性交付包里明确列为“需要人类执行或确认”，不要硬塞给智能体。

## 不记录为正式产物的内容

- 中间迭代流水；
- 智能体内部工作记录；
- 完整聊天记录；
- runtime 数据；
- 生成缓存。
